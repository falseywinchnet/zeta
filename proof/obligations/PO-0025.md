# PO-0025 — simultaneous translation operator in coordinates

Frame: S06

Statement: for the actual coordinate `y=-s`, every translated endpoint curve
`a ↦ y(u+a)` has velocity `Q(y(u+a))`.  For an ordered triple of original
endpoints, the induced derivatives of the maintained coordinate `Lambda` and
`delta` are exactly `coordinateTLambda` and `coordinateTDelta`; the maintained
`coordinatePsi` is `Lambda + (d/da Lambda)/Lambda` along this literal curve.

Quantifiers and domains: every `u<v<w` and every real translation parameter
`a`.  All coordinate points are in the actual range by construction.

Proof: ordinary one-variable composition gives each endpoint velocity.  The
quotient rule differentiates coordinate chord slopes, `Lambda`, and `delta`.
The `delta` calculation retains the term
`-Q(p) * (2-Q''(p))`, so the derivative of the nonconstant coefficient `Q(p)`
is not discarded.

Formalization:
`PF4.SimultaneousCoordinateTranslation.actual_simultaneousCoordinateTranslation`,
with generic chain-rule components in the same module.

Non-vacuity: strict original endpoint order is transported by the globally
strict coordinate map, making both chord denominators nonzero.  No derivative
value or coordinate surjectivity is assumed.

Counterexample condition: an ordered actual endpoint triple and real
translation parameter for which an endpoint velocity, `T Lambda`, `T delta`,
or the resulting `Psi` identity differs from the maintained formula.

Status: `FORMALLY_PROVED`.
